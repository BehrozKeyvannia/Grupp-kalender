KLASS: EventManager

Beskrivning:

 Klassen EventManager är den som ska hålla koll på alla events som skapas och ge ett ID till varje objekt av klassen som skapas. Den klassen ansvarar helt och hållet för alla events och ifall det ska ske någon ändring eller radering av ett event, sköts detta via EventManager. Ingen annan klass får ha någon form av ansvar över inkommande events eller liknande, detta för att minska kohesion och koppling. Klassen har en hashmap för användarnamn som är länkad med anknytande event till användarnamnet. Hashmappen kallas hmap. Likaså finns det en lista på alla events som kallas eventList.

Instansvariabler: hmap och eventID

Metoder: 

Event: createEvent()  == En metod som skapar ett event och metoden kräver inparameterar för att instansiera ett event. Inparametrar är username, titel, start-tid, slut-tid. Metoden har även en räknare som ger ett unikt nummer till varje Event får, varje gång den skapas. Detta unika nummer kallas vi för ID. Varje gång ett event har skapats, kommer räknaren att incrementeras. 

void : addEvent() == Metoden tar existerande objekt av klassen Event och lägger till i hashmappen (hmap). 

Event: getEvent() == Metoden tar namnet på användaren och det ID till eventet som man vill ha. Metoden hämtar Eventet och returnerar det som ett ”Event”.

int : getEventID() == Metoden hämtar ID numret till Eventet som önskas. (vad behövs för inparameter här?)
